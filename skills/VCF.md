# 目标
交付一份可追溯的图像视觉认知分析，说明观众如何逐步看懂画面、当前效果由何而来，以及最有价值的改进方向。

# 工作流程
1. **输入分析**：确认图像清晰度、画面类型和分析目标；不预设主题、人物身份或故事背景。
2. **渐进观察**：模拟首次观看，按视觉显著性依次记录第一眼、第二眼、第三眼及后续注意力落点；每次只依据此前已看见的信息更新判断。
3. **假设收敛**：针对主体和场景提出第一、第二、第三假设，并随新观察修正，直至形成最可信的理解。
4. **效果验证**：从构图、色彩、光影、空间、信息密度和叙事六个角度，检验前述的注意力、场景和情绪判断。
5. **校验与输出**：归纳造成当前效果的关键原因，提出按优先级排序、可执行且能验证的修改建议。

# 关键规则
- 必须模拟普通用户的首次观看过程：先局部、后整体；禁止一开始就以“已知整张图”为前提下结论。
- 每次注意力转移都要说明触发因素，例如尺寸、对比、色相、亮度、人物视线、方向线、位置、文字或运动感。
- “观察到的事实”“当前推断”和“尚未确认的信息”必须分开写；不把推测当作事实。
- 主体判断需要比较元素的面积、对比、位置、清晰度和叙事重要性，而不是只按单一特征判断。
- 信息密度以约 5° 注视范围为估计单位，说明其中竞争注意力的元素、可能的认知负荷及其对理解的影响；无法可靠量化时给出定性判断和依据。
- 情绪分析须分别覆盖人物表情、姿态、色彩、光影和环境，并判断它们是否一致、互补或冲突。
- 用户的分析目标优先于默认框架；若图片模糊、裁切严重或关键区域不可见，应说明限制，不虚构细节。

# 质量检查
- 视觉阅读是否按时间顺序展开，并且每一步都保留“已知信息—新观察—新理解”的推理链。
- 结论是否能回溯到具体可见元素与专业验证项，而非抽象评价。
- 是否覆盖主体、场景、情绪、信息密度和六项专业验证；不适用的项需说明原因。
- 优化建议是否具体、按影响排序且能指导实际修改。
- 若无法从图像可靠判断，应保留不确定性，并向用户确认分析重点或索取更清晰、完整的图片。
