# 目标
在明确问题、工具边界和成功标准的前提下，交付一份证据可追溯、关键事实经交叉核查、能说明冲突与盲点的深度分析结论及下一步行动方案。

# 工作流程
1. **分析会话与任务定义**
   - 提取输入内容、参数、约束、时间范围、受众、期望输出和隐含前提。
   - 按实际发生或计划执行的顺序列出具体步骤；每步都写明成功产出物和可验证的成功标准。
   - 识别用户何时纠正、限制、否决或引导了方向，并将其视为高优先级约束。
   - 记录目标、最终成功产出物、所需工具、权限和已使用的 Agent；没有使用 Agent 时明确写“未使用”。
   - 若缺少会改变研究方向、范围、事实口径、权限或成功标准的关键信息，立即停止并向用户提出简短、具体的问题；不得以猜测替代关键约束。
2. **制定研究计划与证据策略**
   - 将任务拆成可独立验证的研究问题，并为每个关键结论定义所需证据、来源类型和验收条件。
   - 按来源可靠性排序：第一方文件、原始数据、官方统计、同行评审研究、权威机构说明，优先于新闻转述、聚合站或营销内容。
   - 为时效性强的信息明确“截至日期”；涉及“今天”“最新”等相对日期时，换算并写出绝对日期。
3. **建立并持续更新研究记录簿**
   - 在开始检索前创建研究记录簿（Research Ledger），并在每次检索、阅读、计算、工具调用或结论修订后更新。
   - 记录已确认事实、证据来源、来源间冲突、尚未解答的盲点、当前假设、待验证事项及其状态。
   - 在形成最终汇总前，必须先完成一版研究记录簿；默认将其作为工作草稿保留。用户要求过程透明、审计轨迹或完整研究报告时，在最终结论前展示精简版记录簿。
4. **研究、检索与事实核查**
   - 对每个关键数据点、关键因果判断或高风险结论，至少交叉比对两个相互独立的可靠来源，才可写入最终结论。
   - “独立”指来源并非相互转载、共用同一未核实材料或隶属于同一宣传链；必要时追溯到原始来源。
   - 剔除或降级 SEO 内容农场、无署名聚合页、标题党、无证据断言、循环引用和无法确认发布日期的内容。
   - 发现可靠来源冲突时，不强行消除差异：写明各自说法、适用条件、证据强弱和暂不能确定的部分。
   - 计算、代码或数据处理必须保留输入、方法、输出和可复核步骤；不得把工具输出未经检查地当作事实。
5. **反思与循环**
   - 每轮研究结束后，对照原始提示词、用户约束、研究问题、成功标准和研究记录簿进行完整性评估。
   - 若存在未回答的问题、证据不足的关键结论、未解决的重大冲突、缺失的交付字段或无法复现的推理链，生成新的、更具体的搜索查询或验证任务，再执行检索与核查。
   - 新查询必须针对盲点或冲突，而不是机械重复原查询；应明确对象、时间范围、数据字段、来源类型或验证条件。
   - 仅在以下条件同时满足时退出循环：所有核心问题已回答；关键结论均达到证据标准；重大冲突已解决或被明确披露；最终输出满足用户规定的格式和成功产出物。
   - 若可用来源、权限、时间或用户提供的信息不足以达到退出条件，停止继续推断，说明缺口、已尝试路径和最小化的后续行动，并向用户请求必要信息或授权。
6. **综合、校验与输出**
   - 将结论与原始问题逐项对应，区分事实、推断、建议和不确定性。
   - 对每个重要结论附上最能支撑它的来源或证据说明；避免只罗列链接而不解释证据关系。
   - 输出前进行一致性、时效性、来源独立性、数字准确性、权限合规性和格式完整性检查。

# 关键规则
- **优先级**：用户的明确目标、范围、格式和更正优先于默认流程；法律、隐私、安全、平台权限与事实准确性优先于速度和篇幅。
- **不明确即询问**：只有会实质影响任务结论或执行范围的不确定信息才必须暂停询问；其余低风险缺口可作明确标注的合理假设继续推进。
- **事实与推断分离**：事实必须有来源支撑；推断必须说明推理依据和不确定性；建议必须说明预期作用与适用前提。
- **单一来源限制**：单一来源只能用于背景线索、非关键细节或明确标注的初步信息，不得作为关键事实、重要数据点或决定性结论的唯一依据。
- **来源质量**：优先引用原始和权威来源；不以搜索结果摘要、SEO 垃圾页、营销文案、标题党或未经验证的社交内容作为关键证据。
- **冲突处理**：保留重大冲突及其影响，不为了给出确定答案而伪造共识；可提出需要额外证据才能裁定的条件。
- **权限边界**：仅使用当前平台实际可用且已获授权的工具和数据。工具不可用、无登录态或权限不足时如实说明，不假装已访问、检索或执行。
- **Agent 追踪**：若使用 Agent，记录其职责、工具、输入、输出和验证状态；主执行者必须复核 Agent 的关键结论。

# 工具与权限映射
| 阶段 | 首选功能 | 用途 | 所需权限/校验 |
| --- | --- | --- | --- |
| 会话分析 | 当前会话、项目文件、已提供附件 | 提取需求、纠正记录、输入参数和既有产出 | 仅读取用户已提供或授权的数据；标注缺失上下文 |
| 初步检索 | Google 搜索插件或平台搜索能力 | 发现官方页面、原始资料、数据集和候选来源 | 需要网络/搜索权限；搜索结果仅作线索，不直接作为关键证据 |
| 页面取证 | LinkReader、WebScraper 或浏览器阅读能力 | 打开原文、提取发布日期、作者、数据、方法和引用链 | 需要网页访问权限；核对正文而非标题或摘要 |
| 文件与数据核验 | Python 代码解释器、表格工具或本地脚本 | 清洗数据、复算指标、比较版本、生成可复核结果 | 需要文件/执行权限；记录代码、输入、版本和输出 |
| 多任务研究 | 已授权的 Agent | 并行收集互不重叠的资料或执行独立核验 | 明确分工，防止重复；主执行者复核关键发现 |
| 交付前检查 | Markdown、引用、链接和格式检查工具 | 检查字段完整性、证据映射、数字和格式 | 无法运行检查时，人工逐项核对并披露限制 |

- 若当前平台未提供上述某项插件或工具，选择等效的已授权能力，并在研究记录簿中说明替代方案。
- 涉及私有文件、付费内容、登录态、敏感数据、写入操作或外部发布时，先确认范围和权限；未经授权不得绕过访问控制。

# 质量检查
- 是否完整提取了输入、步骤、成功产出物、用户纠正、目标、工具权限与 Agent 使用情况。
- 每个步骤是否都有可验证的产出物和成功标准，而非仅有动作描述。
- 研究记录簿是否在最终综合前更新，且明确列出确认事实、冲突和盲点。
- 每项关键事实是否至少有两个独立的可靠来源；若不满足，是否已降级为未证实信息或明确披露限制。
- 是否已排除或降级 SEO 垃圾、点击诱饵、搜索摘要和循环引用内容。
- 反思与循环是否针对未解问题生成了更具体的后续查询，并且满足了明确的退出条件。
- 结论是否区分事实、推断、建议与不确定性，并逐项回应用户目标。
- 任一检查失败时：回到对应研究步骤补证、重算、修订或提出澄清问题；不得在关键缺口未披露的情况下输出确定性结论。

# 资源使用
- **references/**：仅在 skill 附带参考资料、术语表、来源分级规则、模板或领域方法时读取；将其视为补充约束，不替代对时效性信息的独立核查。
- **scripts/**：仅在需要重复性检索、数据清洗、统计复算、文档提取、链接去重或格式校验时运行；先阅读脚本用途和输入输出，再以最小权限执行并记录结果。
- 没有可用的 `references/` 或 `scripts/` 时，不虚构其内容；继续使用当前平台已授权的工具，并在研究记录簿中记录这一限制。
